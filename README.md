# 💸 چیو کجا خرج کردم؟!

> وقتی آخر ماه نگاه به حساب بانکیت می‌کنی و می‌گی: «این پول‌ها کجا رفتن؟!» 😱

---

## 🎯 این اپ چیکار می‌کنه؟

**چیو کجا خرج کردم** یه اپ مدیریت مالی خانوادگیه که کمک می‌کنه دقیقاً بدونی پولت کجا رفته — و دیگه آخر ماه شوکه نشی! 😅

---

## ✨ قابلیت‌ها

### 👨‍👩‍👧‍👦 مدیریت خانوادگی
همه اعضای خانواده زیر یه سقف! هر کس با اکانت خودش وارد میشه، هزینه‌هاشو ثبت می‌کنه و همه می‌تونن کل تصویر مالی خانواده رو ببینن.

```
بابا → قبض برق ✅
مامان → خرید هفتگی ✅
پسر → ... 🍕🍕🍕 (باز پیتزا؟!)
```

---

### 💳 مدیریت کارت‌های بانکی
چند تا کارت داری؟ ده تا؟ صد تا؟ 😂 همشون رو اینجا اضافه کن. موجودی هر کارت به‌صورت جداگانه دنبال میشه و با هر تراکنش آپدیت میشه.

---

### 📊 انواع تراکنش

| نوع | توضیح | آیکون |
|-----|-------|-------|
| خرج | پول دادی و رفت | 💸 |
| درآمد | پول اومد (نادر ولی اتفاق می‌افته!) | 🤑 |
| کارت به کارت | از این جیب به اون جیب | 🔄 |

---

### 🏷️ دسته‌بندی و تگ‌گذاری
هر هزینه رو دسته‌بندی کن — خوراک، حمل‌ونقل، تفریح، قبوض... حتی می‌تونی تگ بزنی. مثلاً «سفر نوروز ۱۴۰۴» رو جداگانه ببینی چقدر آب رفت! 🏖️

---

### 📈 گزارش‌گیری
می‌خوای بدونی بیشتر پولت کجا میره؟

- 📅 فیلتر بر اساس بازه زمانی (این ماه، ۳ ماه اخیر، سال گذشته، دلخواه)
- 📂 گروه‌بندی بر اساس دسته‌بندی یا تگ
- 📄 خروجی PDF بگیر و برو به حسابدار نشون بده (یا از بقیه مخفیش کن 🤫)

---

### 🔔 یادآور قبوض
قبض برق، گاز، تلفن... همه رو ثبت کن، اپ موقع سررسید بهت یادآوری می‌کنه. دیگه بهانه «یادم رفت» نداری! 😤

---

### ☁️ سینک ابری
همه داده‌ها روی Supabase ذخیره میشن. گوشیت عوض شد؟ هیچی گم نمیشه! آفلاین بودی؟ وقتی اینترنت برگشت همه چیز سینک میشه. 🪄

---

### 🌙 تم تاریک
چون چشمامون ارزش دارن. 😎

---

### 📆 تقویم شمسی
همه تاریخ‌ها جلالی/شمسی هستن — چون ما ایرانی‌ایم و می‌دونیم ۱۴۰۴/۰۱/۱۵ یعنی چی، ولی ۲۰۲۵-۰۴-۰۴ رو باید حساب کنیم! 🗓️

---

## 🛠️ بخش فنی (مختصر!)

این پروژه **کاملاً اوپن سورس** هست و با خیال راحت می‌تونی کدش رو بخونی، فورک کنی، یا بهش کمک کنی. 🤝

| مورد | جزئیات |
|------|---------|
| زبان اصلی | Java |
| معماری | MVVM + Room |
| بک‌اند | Supabase (PostgreSQL) |
| پلتفرم | Android |
| Build system | Gradle (Kotlin DSL) |

---

## 🔢 ورژن‌بندی

فرمت: `v{اصلی}.{بزرگ}.{باگ‌فیکس}`

```
v1 . 2 . 13
│    │    └─ شماره باگ‌فیکس — با هر رفع اشکال یه واحد بالا میره
│    └────── آپدیت بزرگ — وقتی یه قابلیت مهم اضافه میشه
└─────────── ورژن اصلی — تغییرات کلی در ساختار برنامه
```

**مثال:**
- `v1.2.13` ← ورژن اصلی ۱، آپدیت بزرگ دوم، سیزدهمین باگ‌فیکس
- `v1.2.14` ← یه باگ دیگه پیدا و فیکس شد 🐛
- `v1.3.0` ← یه قابلیت جدید بزرگ اضافه شد 🎉
- `v2.0.0` ← اپ از نو بازنویسی شد (انشاءالله نرسیم! 😂)
